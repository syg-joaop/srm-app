---
description: SRM
---

Quando for criar código novo sempre procure componentes reutilizaveis no projeto, se não existir crie-os, o importante é nunca repetir código, sempre reaproveitar.
Sempre respeite o padrão de visual do projeto, nunca saia do padrão já estabelecido.
Nunca escreva comentários no código.
Os nomes dos arquivos tem que serem sempre em PORTUGUÊS do Brasil.
Não utilize Composition API, faça sempre com Options.
